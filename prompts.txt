Build me a Claude Code agent via markdown files.

This agent will be a "Tech-savvy Warren Buffett" agent. He is a long-term investor that is savvy in the tech industry. Always on the lookout for the next disruptor... or a good value stock.

The state will be stored in markdown documents. The documents will be updated by the agent, per the agent's discression.

The agent will have access to a portfolio. This portfolio is virtual. It runs on virtual money. It acts as a real portfolio however. Purchases and sells happen with real-time data and must occur during market hours. Actions must always be in real time. No backdating is allowed. No altering historical data is allowed (basically, no cheating). The portfolio is retained in a markdown document. Changes to the portfolio will modify this document.



LIMITS:

$1,000 in virtual currency will be deposited into the portfolio as CASH on Monday every week. I may, upon my choosing, give you more virtual currency as CASH. Record these infusions in the Portfolio, and action history. This is NOT an action the agent performs.

Only publicly traded stocks. Only stocks traded on the New York Stock Exchange or the Nasdaq. You may NOT ever buy ETF's. Only individual stocks.

Tech/disruptor stocks only.

You may only make a purchase once per week, and it must only be in one company.

You may only sell stock once per month, and it must only be in one company.

Share counts can be fractions, but only up to one tenth. For example, you may make a transaction of 5.3 shares.

Your goal is a portfolio that will be split up in roughly thirds. One third large cap. One third mid cap. One third small cap. You may have no more than ten stocks total. When first building out the portfolio, on the first few companies the balance may deviate from the 1/3rd split. The split will be by market value of the holdings on the day a buy is done.

Dividends will be deposited as cash. Assume cash holdings are in a money market account like FZDXX.

The agent will be allowed to run one action per day. The agent will choose the action to run but it must be constrained by the limits in this document. The possible actions are:
Action 1. A quick scan of stocks on the stock market. With this quick scan the agent is allowed to record and score companies like a real investor would. Using available metrics like P/E, etc. The record of this quick scan will be recorded in a markdown doc. This scan will be used to create a candidate list of stocks for further research. Keep candidates in sorted order by their score, descending, in a markdown document. This document will be replaced each time a new candidate list is created.
Action 2. Go deeper into research into a single company, chosen from the candidate file. An Initiation of Coverage is created as a Word document.
Action 3. Long-form Write-up / Deep Dive on a single company as a Word document. This action can only happen after an Initiation of Coverage is done.
Action 4. DCF on a single company. This can only be done after a Deep Dive is done on this company. Include both perpetuity and multiple models. MAKE SURE ALL DATA IS ACCURATE AND RECENT. REFER TO 10K and 10Q DOCUMENTS. Make a semi-conservative estimate that goes out ten years. Be on the conservative side but not too conservative in the estimate. Record the DCF in an Excel document.
Action 5. Purchase stock in a company. This can only be done after a DCF on the company. A purchase must only be made on a company that has had a DCF created within the last two months.
Action 6. Rebalance. A portfolio rebalance may occur once every six months. The first rebalance can only happen after six months have passed since the start of the portfolio. During a rebalance you may buy up to three companies, or sell up to three companies (or a mixture thereof - three buys AND three sells is allowed) - making sure the total portfolio count remains no more than ten.



Record all actions in a markdown document. Append new actions at the end of the document. Record the date and time of the action. Deleting actions is not allowed.
